// What is RHCSA

The exam that
proves you can
run Linux.

The Red Hat Certified System Administrator (RHCSA) is a performance-based exam. You are given a live system and a set of tasks — there are no multiple choice questions, only real commands.

This guide covers everything from your classwork: Vim, SSH, file management, users, permissions, storage, LVM, containers, networking, SELinux, and more.

Every chapter here is directly from lecture notes, structured for fast revision and exam performance.

⏱️
3 Hours, 2 Nodes
The RHCSA exam (EX200) runs for 3 hours on 2 virtual machines. You must complete hands-on tasks on both nodes — Node 1 and Node 2.
🎯
Score 210 / 300
Passing score is 210 out of 300. Tasks are graded automatically after the exam. Partial credit may be given.
🔑
Key Exam Topics
Root password reset, LVM, SELinux, NFS autofs, Containers (Podman), scheduling tasks, network config, user management.
// Exam Tips
How to score
maximum marks.
01
Always verify after every task
After completing a task, verify it works. Use systemctl status, df -h, mount -a, or podman ps to confirm your work before moving on.
02
Root password reset is critical
This task is nearly always on the exam. Master every step — rd.break, mount remount,rw, chroot, passwd, touch /.autorelabel — especially the autorelabel step.
03
loginctl enable-linger for Podman
For rootless containers that must start at boot — not just at login — you must run loginctl enable-linger. Forgetting this loses 25% of container marks.
04
XFS cannot be reduced
Only EXT family filesystems (ext2/3/4) can be reduced with resize2fs. If the filesystem is XFS, you can only extend it — never reduce. This is a classic exam trap.

CH-1
Vim & SSH
Basic shell commands·Beginner
▶

Shell Basics

  • Bash = Bourne Again Shell
  • ~ = home directory  |  root = administrator
  • Command syntax: command [option/argument] [target/path]

Getting Help

command --help        # Show all available options
man command           # Display the manual page

Basic Commands

ls          # List directory contents
ls -l       # Long listing format
ls -a       # Include hidden files
ls -t       # Sort by modification time (newest first)
touch file  # Create blank file
mkdir dir   # Create blank directory
rm file     # Remove file
rmdir dir   # Remove blank directory (only if empty)
rm -rf dir  # Force remove file or directory (dangerous!)
cat file    # Read file content
head file   # First 10 lines  |  head -n 5 file → first 5 lines
tail file   # Last 10 lines   |  tail -f file → real-time follow
more file   # Gradually read whole file
echo "txt"  # Print text / insert into file

SSH — Secure Shell

Used to take a remote connection to any system.

ssh username@hostname_or_ip

Case 1: Passwordless Login (Key-based)

ssh-keygen                              # Generate key pair
ssh-copy-id user@remote_hostname        # Copy key to remote
ssh user@remote_hostname                # Login without password

Case 2: Passphrase Key

ssh-keygen           # Enter passphrase when prompted
ssh-copy-id user@host
ssh user@host        # Now asks for passphrase each time

Case 3: SSH-Agent (Skip passphrase)

eval $(ssh-agent)               # Start ssh-agent
ssh-add .ssh/private_key_name   # Add key to agent
ssh user@host                   # Login without passphrase

Non-Default Key (-f)

ssh-keygen -f .ssh/mykey                      # Create named key
ssh-copy-id -i .ssh/mykey.pub user@hostname   # Copy it
ssh -i .ssh/mykey user@hostname               # Use it

Shell Expansions

Users & Groups
useradd, usermod, sudo, shadow·Intermediate
▶

Types of Users

TypeUID RangeDescription
Superuser (root)0Full system control
System User1–999Used by services/daemons
Regular User1000–60000Normal login users

Check UID/GID: id username

/etc/passwd Structure

username : x : UID : GID : /home/dir : /bin/shell
# x = password stored in /etc/shadow

Create & Manage Users

useradd username        # Create user
passwd username         # Set password
useradd --help          # View all options

usermod — Modify Existing User

usermod -l newname oldname          # Rename user
usermod -g groupname username       # Change primary group
usermod -aG groupname username      # Add secondary group (-a is crucial!)
usermod -aG group1,group2 username  # Add multiple groups
usermod -d /new/home username       # Change home dir (no move)
usermod -d /new/home -m username    # Change home dir + move files
usermod -s /bin/bash username       # Change login shell
usermod -L username                 # Lock account
usermod -U username                 # Unlock account
usermod -u 3005 username            # Change UID
usermod -g 5000 username            # Change GID
usermod -e YYYY-MM-DD username      # Set account expiry
usermod -c "Full Name" username     # Change comment/full name
⚠️ Always use -a with -G. Without -a, existing groups get overwritten!

groupmod — Modify Groups

groupmod -n newgroup oldgroup     # Rename group
groupmod -g newGID groupname      # Change group GID

Group info stored in: /etc/group

SUDO Configuration

# Edit sudoers safely:
visudo    # or edit /etc/sudoers

# Give sudo to a user:
username ALL=(ALL) ALL

# Give sudo to a group:
%groupname ALL=(ALL) ALL

# Skip password:
username ALL=(ALL) NOPASSWD:ALL

Switch Users (su)

su username        # Switch without loading env
su - username      # Switch WITH full env & filesystem
su - root          # Switch to root (needs root password)
su -               # Same as above
sudo -i            # Switch to root using YOUR password

Password Policy — /etc/shadow

# Fields: user:encrypted_pass:last_change:min:max:warn:inactive:expiry
chage username          # Interactive policy editor
chage -l username       # List current policy
date -d "240 days"      # Calculate date 240 days from today

Future user defaults: /etc/login.defs

CH-4
Access Control to Files (DAC)
chmod, chown, SUID, SGID, sticky bit, umask·Intermediate
▶

Permissions Overview

View permissions: ls -l filename

-rwxrwxrwx
 │││││││││
 ││││││└└└── Others (r,w,x)
 │││└└└───── Group  (r,w,x)
 └└└──────── User   (r,w,x)

Symbolic Method

# Syntax: chmod [who][+/-/=][permission] file
# who: u=user, g=group, o=others, a=all

chmod u+x file       # Add execute for user
chmod g-w file       # Remove write from group
chmod o=r file       # Set others to read only
chmod a+x file       # Add execute for everyone
chmod ugo+rwx file   # Full permissions for all

Octal Method

PermissionValue
read (r)4
write (w)2
execute (x)1
all (rwx)7
chmod 755 file   # rwxr-xr-x (user=7, group=5, others=5)
chmod 777 file   # rwxrwxrwx (full for all)
chmod 644 file   # rw-r--r-- (common for files)

Changing Ownership

chown user file                  # Change user ownership
chown user:group file            # Change user + group
chgrp groupname file             # Change group only

Special Permissions

SUID (Special User — runs as file owner)

chmod u+s file        # Set SUID (symbolic)
chmod u-s file        # Remove SUID
chmod 4744 file       # Set SUID (octal: 4 prefix)
chmod 0744 file       # Remove SUID

SGID (Special Group — new files inherit group)

chmod g+s path        # Set SGID
chmod g-s path        # Remove SGID
chmod 2744 file       # Set SGID (octal: 2 prefix)

Sticky Bit (Users can't delete other's files)

chmod o+t path        # Set sticky bit
chmod o-t path        # Remove sticky bit
chmod 1744 file       # Set sticky bit (octal: 1 prefix)

Default Permissions — umask

umask                # Check current umask value
umask 027            # Set temporary umask

# Max file permission = 666, max dir = 777
# Default perm = Max - umask

# User-specific: edit ~/.bashrc → add "umask 027" at end
# Global:        edit /etc/profile → add "umask 027" at end
💡 User settings take priority over global settings.

Processes, Monitoring & Scheduling
ps, top, kill, nice, crontab, at, tuned·Intermediate
▶

Process Types

  • Foreground — Attached to terminal. Ex: vim file.txt
  • Background — Runs behind. Ex: firefox & → shows [jobID] PID

Job Control

command &        # Start in background
jobs             # List shell jobs
fg %1            # Bring job 1 to foreground
bg %1            # Resume stopped job in background
Ctrl-Z           # Suspend current foreground process
disown %1        # Remove job from shell's job table

Process Listing

ps                          # Processes in this terminal
ps -l                       # Long format (STAT, PPID, PRI)
ps aux                      # ALL processes, all users
ps -o pid,ppid,uid,cmd      # Custom columns
pgrep firefox               # Find PIDs by name
pidof sshd                  # Get PID of service
pstree                      # Process tree (parent-child)
top                         # Real-time viewer
htop                        # Better real-time viewer

Process States (STAT column)

StateMeaning
RRunning or Runnable
SSleeping (waiting for event)
DUninterruptible sleep (I/O wait)
TStopped or Traced
ZZombie (finished but not reaped)

Signals & Kill

kill <PID>             # SIGTERM (polite request)
kill -9 <PID>          # SIGKILL (force kill, cannot be caught)
kill -SIGSTOP <PID>    # Suspend process
kill -SIGCONT <PID>    # Resume suspended process
kill -l                 # List all signal names
pkill processname       # Kill by name (SIGTERM)
pkill -9 processname    # Force kill by name
killall processname     # Kill all with that exact name

Load Monitoring

uptime                  # Shows load averages: 1, 5, 15 min
lscpu                   # CPU count (check CPU(s): field)
# If load_avg / CPU_count > 1 → system is overloaded

Process Priority — nice & renice

Range: -20 (highest priority) to +19 (lowest). Default = 0.

nice -n 10 command          # Run with lower priority (+10)
renice -n -5 -p 12345       # Change priority of running process
ps -l                        # View NI (nice) column
Only root can set negative nice values (higher priority).

Tuned — System Optimization

tuned-adm active                          # Check active profile
tuned-adm recommend                       # Get recommended profile
tuned-adm profile virtual-guest           # Set a profile
systemctl status tuned.service            # Check if installed/running
# Config: /etc/tuned/tuned-main.conf
# Profiles: /usr/lib/tuned/

Scheduling — at (One-Time)

at 07:40                  # Schedule interactive task
at now + 5 minutes        # Relative time
at tomorrow               # Tomorrow same time
atq                       # View pending jobs
atrm 1                    # Remove job ID 1

Scheduling — crontab (Recurring)

# Syntax: M H D_M Month D_W COMMAND
# M=minute(0-59) H=hour(0-23) D_M=day-of-month(1-31)
# Month(1-12) D_W=day-of-week(0-7, 0&7=Sunday)

crontab -e           # Edit cron jobs
crontab -l           # List current jobs
crontab -r           # Remove ALL jobs (careful!)
crontab -eu student  # Edit another user's crontab (as root)

# Example: run every minute:
* * * * * echo "hello" >> /tmp/cron.txt

# System-wide crontab: /etc/crontab (has extra USER field)
* * * * * student echo "hello" >> hello.txt

# Access control:
/etc/cron.allow   # Only listed users can use crontab
/etc/cron.deny    # Listed users CANNOT use crontab

Manage Basic Storage & Partitions
fdisk, mkfs, mount, fstab, swap·Intermediate
▶

Check Storage

lsblk               # List block devices + partitions
lsblk -fp           # Include filesystem type + UUID
df -h               # Disk usage of mounted filesystems

Partition Schemes

SchemeMax PartitionsMax SizeType
MBR (dos)15 (4 primary + 11 extended)2 TBOldest, BIOS/UEFI
GPT1288 ZettabytesNew, UEFI only

Create Partition with fdisk

fdisk /dev/sdb       # Open disk for editing
# Inside fdisk:
n     → New partition
p     → Primary (default, just press Enter)
Enter → Accept default first sector (ALWAYS)
+2G   → Set size (e.g., 2 GB)
p     → Print/verify partition table
w     → Write changes and exit

partprobe            # Inform kernel of new partition

Format & Mount

mkfs.xfs /dev/sdb1       # Format as XFS
mkfs.ext4 /dev/sdb1      # Format as EXT4

mkdir /mnt/data1                    # Create mount point
mount /dev/sdb1 /mnt/data1          # Temporary mount
df -h                               # Verify mount

Permanent Mount — /etc/fstab

vim /etc/fstab
# Add line:
/dev/sdb1  /mnt/data1  ext4  defaults  0  0
# Fields: device | mountpoint | filesystem | options | dump | fsck

systemctl daemon-reload    # Reload systemd
mount -a                   # Test fstab (no output = success!)
df -h                      # Verify

Unmount & Delete Partition

umount /mnt/data1        # Unmount
lsof /mnt/data1          # Check if partition is busy
fuser -m /mnt/data1      # Another way to check

# Delete: fdisk /dev/sdb → d → p → w → partprobe

Swap Partition — Memory Code: CFPA

# C - Create: fdisk → n → +1G → t → 82 (MBR) or 8200 (GPT) → w
# F - Format: mkswap /dev/sdXN
# P - Persistent: vim /etc/fstab → /dev/sdXN swap swap defaults 0 0
# A - Activate: swapon -a

swapon --show    # Verify swap is active
lsblk            # Double-check

# Set swap priority in fstab:
/dev/vdc1  swap  swap  pri=10  0  0   # Higher = used first
CH-10
LVM — Logical Volume Manager
PV → VG → LV, extend, reduce, delete·Advanced
▶

LVM Hierarchy

Physical Volume (PV) → Volume Group (VG) → Logical Volume (LV)

Part A: Create LVM (Full Steps)

# Step 1: Create LVM-type partition
fdisk /dev/sdb
# n → Enter → +1G → t → l (list types) → 8e (LVM for MBR) → w
partprobe

# Step 2: Create Physical Volume
pvcreate /dev/sdb1
pvdisplay              # Verify

# Step 3: Create Volume Group
vgcreate myvg /dev/sdb1                   # Basic
vgcreate -s 16M myvg /dev/sdb1           # With custom PE size (default 4M)
vgdisplay                                  # Verify

# Step 4: Create Logical Volume
lvcreate -n mylv -L 5G myvg               # By size
lvcreate -n mylv -l 100 myvg              # By PE count

# Step 5: Give filesystem
mkfs.xfs /dev/myvg/mylv
# or: mkfs.ext4 /dev/myvg/mylv

# Step 6: Mount point + fstab
mkdir /mnt/data
vim /etc/fstab
# /dev/myvg/mylv  /mnt/data  xfs  defaults  0  0
mount -a                  # Test (no output = success)
df -h                     # Verify

Part B: Extend LVM

# Extend by size:
lvextend -L +1G /dev/myvg/mylv

# Extend LV AND filesystem in one command:
lvextend -L +500M -r /dev/myvg/mylv      # -r = auto resize FS

# Extend filesystem manually:
xfs_growfs /dev/myvg/mylv                # For XFS
resize2fs /mnt/data                       # For EXT family

# If VG is full — extend VG first:
pvcreate /dev/sdc1
vgextend myvg /dev/sdc1
lvextend -L +5G -r /dev/myvg/mylv

Part C: Reduce LVM

🚫 XFS CANNOT be reduced! Only EXT family (ext2/3/4) can be reduced. This is a critical exam point!
# Safe steps for reducing EXT volume:
umount /mnt/data                          # 1. Unmount
fsck /dev/myvg/mylv                       # 2. Check filesystem
resize2fs /dev/myvg/mylv 3G               # 3. Shrink filesystem
lvreduce -L 3G /dev/myvg/mylv            # 4. Shrink LV
mount -a                                  # 5. Remount

Part D: Delete LVM (Reverse Order)

umount /mnt/data              # 1. Unmount
vim /etc/fstab                # 2. Remove fstab entry
systemctl daemon-reload       # 3. Reload
lvremove /dev/myvg/mylv       # 4. Remove LV
vgremove myvg                 # 5. Remove VG
pvremove /dev/sdb1            # 6. Remove PV

Boot Process & Control Services
systemctl, targets, root password reset·Advanced
▶

systemd Service Management

systemctl start httpd             # Start (temporary)
systemctl stop httpd              # Stop
systemctl status httpd            # Check status
systemctl enable httpd            # Enable at boot
systemctl disable httpd           # Disable at boot
systemctl enable httpd --now      # Enable + start immediately
systemctl restart httpd           # Restart (stop + start)
systemctl reload httpd            # Reload config (no stop)
systemctl reload-or-restart httpd # Let system decide
systemctl mask httpd              # BLOCK from starting at all
systemctl unmask httpd            # Unblock

Targets (Runlevels)

systemctl get-default                        # Check default target
systemctl set-default multi-user.target      # Set CLI as default (needs reboot)
systemctl isolate multi-user.target          # Switch NOW (temporary)
systemctl isolate graphical.target           # Switch to GUI NOW
systemctl list-dependencies httpd            # View dependencies

Boot Process (6 Stages)

  1. BIOS/UEFI — Hardware check, loads bootloader
  2. GRUB2 — Shows OS menu, loads Kernel + initramfs
  3. Kernel — Detects hardware, starts systemd (PID 1)
  4. initramfs — Temporary root FS, loads essential drivers
  5. systemd — Reads default target, starts services
  6. Login Screen — CLI or GUI based on target

Root Password Reset (EXAM CRITICAL!)

🔑 This is THE most important RHCSA exam question. Follow these steps exactly.
# Step 1: Reboot
reboot

# Step 2: Stop GRUB countdown
# Press Up/Down arrow keys

# Step 3: Select boot entry and press 'e' to edit

# Step 4: Find line starting with "linux", press End key

# Step 5: Add at end of that line (after space):
rd.break
# For RHEL 9.3: also delete "console=tty0"

# Step 6: Press Ctrl+X to boot
# You'll get emergency shell. Filesystem is READ-ONLY.

# Step 7: Remount as read-write
mount -o remount,rw /sysroot

# Step 8: Enter real root filesystem
chroot /sysroot

# Step 9: Change password
passwd

# Step 10: CRITICAL - Create SELinux relabel file
touch /.autorelabel    # Without this → system WON'T boot!

# Step 11: Exit and reboot
exit
exit

CH-13
SELinux — Security-Enhanced Linux
Contexts, ports, booleans, semanage·Advanced
▶

What is SELinux?

SELinux = Mandatory Access Control (MAC). Works at the kernel level using tags (contexts). If process tag and resource tag match → allow. Otherwise → deny.

Check Status & Modes

sestatus           # Check SELinux status
getenforce         # Check current mode
# Config file: vim /etc/selinux/config

# Modes:
# 1 - Enforcing   → SELinux ON (policies applied)
# 2 - Permissive  → SELinux OFF but logs generated
# 3 - Disabled    → SELinux completely OFF

setenforce 1       # Set enforcing (temporary)
setenforce 0       # Set permissive (temporary)

View Context (Tags)

ls -Z file              # File context
ls -dZ /tmp/            # Directory context
ps axZ                  # Process context

Change Context — Temporary

chcon -t httpd_sys_content_t demo.html   # Temporary change

Change Context — Permanent

# 1. Add to SELinux database:
semanage fcontext -a -t httpd_sys_content_t /root/demo.html

# 2. Apply from database:
restorecon -Rv /root/

# View database:
semanage fcontext -l | grep root

Port Context (EXAM IMPORTANT!)

semanage port -l                      # List all port contexts
semanage port -l | grep http          # Filter for http

# Add a port (e.g., allow httpd on port 82):
semanage port -a -t http_port_t -p tcp 82

# Then restart service:
systemctl restart httpd

# Test: curl localhost:82

View SELinux Logs

journalctl | grep sealert    # Find SELinux denials
# Copy the sealert ID and run it as a command for solutions

Booleans

getsebool -a | grep http                 # List http booleans
setsebool httpd_enable_homedirs on       # Set boolean (temporary)
setsebool httpd_enable_homedirs on -P   # Set permanently

▶

firewall-cmd Basics

All firewall commands start with firewall-cmd. Use --permanent to make changes survive reboot.

firewall-cmd --get-services              # List all available services
firewall-cmd --get-default-zone          # Check default zone
firewall-cmd --set-default-zone=public   # Set default zone

# Add/remove service:
firewall-cmd --add-service=http --permanent
firewall-cmd --remove-service=http --permanent

# Add/remove port:
firewall-cmd --add-port=8080/tcp --permanent
firewall-cmd --remove-port=8080/tcp --permanent

# Allow/block source IP:
firewall-cmd --add-source=192.168.1.0/24 --zone=internal
firewall-cmd --remove-source=192.168.1.0/24 --zone=internal

# Reload to apply --permanent changes:
firewall-cmd --reload

# List active rules:
firewall-cmd --list-all
💡 Rule: --get-... to see, --set-... to configure, --add-... to add, --remove-... to delete.

Install & Search

dnf install container-tools     # Install podman
podman -v                        # Verify version
podman search httpd              # Search registry for images
# Registry config: /etc/containers/registries.conf

Images & Containers

podman pull IMAGE_URL             # Download image
podman images                    # List local images
podman rmi IMAGE_URL             # Remove image

# Create + run container:
podman run -d --name webserver IMAGE_URL
# -d = detached (background), --name = container name

podman ps                        # List running containers
podman ps -a                     # All (running + stopped)

podman stop webserver            # Stop container
podman rm webserver              # Remove container
podman rm webserver -f           # Force remove (even if running)

Execute into Container

podman exec -it webserver bash   # Enter container shell
# -it = interactive terminal
cd /var/www/html/                # Navigate to web root
curl localhost                   # Test web content

# Copy file from host to container:
podman cp index.html webserver:/var/www/html/

Port Forwarding

# Must configure at container creation time!
# Delete old container first if needed
podman run -d --name webserver -p 8080:80 IMAGE_URL
# 8080 = host port, 80 = container port
curl localhost:8080    # Test from outside

Volumes — Persistent Data

mkdir /backup
podman run -d --name webserver \
  -v /backup/:/var/www/html/:Z \
  -p 8080:80 IMAGE_URL
# :Z = SELinux context auto-match
# Multiple volumes: repeat -v for each

Build Local Image

# Create containerfile with FROM, RUN, CMD instructions
vim containerfile

podman build -t myimage -f containerfile   # Build
podman run -d --name myapp localhost/myimage:latest

# Push to registry:
podman push localhost/myimage:latest username/registry-name

Container as a Service (Auto-start)

# As ROOT:
cd /etc/systemd/system
podman generate systemd --name webserver --files
systemctl daemon-reload
systemctl enable container-webserver.service --now

Rootless Container — Non-Root User

⚠️ Always SSH as the non-root user directly. su - user won't work for session-based operations.
# Check session: w  (only target user should be active)

# Service location for non-root:
mkdir -p ~/.config/systemd/user
cd ~/.config/systemd/user

podman generate systemd --name myapp --files
systemctl --user daemon-reload
systemctl --user enable container-myapp.service --now
systemctl --user status container-myapp.service

loginctl enable-linger
This makes the container start at system boot, not just at user login.

tar — Archive Files

FlagMeaningExtension
-jbzip2 compression.tar.bz2
-zgzip compression.tar.gz
-Jxz compression.tar.xz
# Create archive:
tar -cvjf backup.tar.bz2 /usr/local
# c=create, v=verbose, j=bzip2, f=filename

tar -cvzf backup.tar.gz /usr/local     # gzip
tar -cvJf backup.tar.xz /usr/local     # xz

# Extract archive:
tar -xvf archive.tar.bz2

find — Locate Files

find / -name newfile                    # Find by name
find / -user jeel                       # Find by owner
find / -group devops                    # Find by group
find / -perm 750                        # Find by permission
find / -perm 750 -type d                # Directories only
find / -perm /g+s                       # Find SGID files
find / -size +30k -size -45k           # Find by size range

# Execute command on found files (-exec):
find / -user jeel -exec cp -pvr {} /root/jeel-files/ \;
# p=preserve perms, v=verbose, r=recursive, {} = file placeholder, \; = end exec

Create & Execute a Shell Script

vim myscript.sh

# Inside file:
#!/bin/bash
find /usr -perm /g+s -size +30k -size -45k > /root/usr-files

# Save and quit: :wq

chmod +x myscript.sh      # Make executable
sh myscript.sh            # Execute
💡 Bonus Trick: sudo !! runs your last command with sudo (when you get permission denied). This is called history expansion.
