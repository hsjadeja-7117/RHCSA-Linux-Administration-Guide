// Study Approach
How to use
these notes.
01
Practice every command
Reading is not enough. Spin up a RHEL or CentOS VM and type every command in these notes. Muscle memory is essential in Linux administration.
02
Understand, don't memorise
Focus on understanding what each command does and why. Use man commandname and command --help to explore options you're unsure about.
03
Always verify your work
After any configuration change, verify it worked. Use systemctl status, df -h, mount -a, or lsblk to confirm changes took effect.
04
Build a lab environment
Use VirtualBox or VMware to create a local RHEL 9 lab. Practice storage tasks, user management, and service configuration in an environment where mistakes are safe.

CH-1
Vim & SSH
Shell basics, remote connections, key auth·Beginner
▶

Shell Basics

  • Bash = Bourne Again Shell
  • ~ = home directory | root = administrator
  • Command syntax: command [option/argument] [target/path]

Getting Help

command --help        # Show all available options
man command           # Display the manual page

Essential Commands

ls              # List directory contents
ls -l           # Long listing format
ls -a           # Include hidden files
ls -t           # Sort by modification time (newest first)
touch file      # Create blank file
mkdir dir       # Create blank directory
rm file         # Remove file
rmdir dir       # Remove blank (empty) directory only
rm -rf dir      # Force remove file or directory — use carefully
cat file        # Read file content
head file       # First 10 lines  |  head -n 5 file = first 5
tail file       # Last 10 lines   |  tail -f file = real-time follow
more file       # Gradually read whole file
echo "text"     # Print text or redirect into file

SSH — Secure Shell

ssh username@hostname_or_ip

Key-Based Authentication

ssh-keygen                              # Generate key pair
ssh-copy-id user@remote_hostname        # Copy public key to remote
ssh user@remote_hostname                # Login without password

SSH-Agent (Cache passphrase)

eval $(ssh-agent)               # Start agent background process
ssh-add .ssh/private_key_name   # Register key with agent
ssh user@host                   # Connect without passphrase prompt

Non-Default Key

ssh-keygen -f .ssh/mykey
ssh-copy-id -i .ssh/mykey.pub user@hostname
ssh -i .ssh/mykey user@hostname

echo — Print & Write Text

echo Hello                      # Print text to terminal
echo "Hello Linux" > file.txt   # Write to file (overwrites)
echo "New line" >> file.txt     # Append to file (keeps existing)

more — Read File Gradually

more filename
# Space → Next page
# q     → Quit

man — Manual Page Navigation

man ls         # Open manual for ls command
# Inside man:
# q       → Quit
# /word   → Search for word
# n       → Jump to next search result

Vim Editor — Essential for Administration

Vim is used to edit configuration files, scripts, and system settings. It has three modes:

Mode Purpose How to Enter
Normal Navigation and commands Default / press Esc
Insert Type and edit text Press i
Command Save, quit, search Press : 
vim filename    # Open file in vim

Essential Vim Commands

Action Command
Enter Insert Mode i
Save :w
Quit :q
Save & Quit :wq
Force Quit (no save) :q!
Delete Line dd
Copy Line yy
Paste p
Search /word

Manage Files from the Command Line
cp, mv, rm, ln, grep, wildcards·Beginner
▶

Linux Filesystem Hierarchy

Everything in Linux starts from / (root). Run ls / to see the top level.

Directory Purpose
/boot Bootloader files and kernel (vmlinuz)
/dev Device files (disk, CPU, keyboard)
/home Home directories for regular users
/root Home directory of root user
/etc Configuration files
/tmp Temporary files (world-writable)
/usr/bin User command binaries
/usr/sbin System administration binaries
/var Variable data — logs, spool, cache
/mnt Temporary mount point
/media Removable devices
/run Runtime process data

Special Path Symbols & Navigation

Symbol Meaning
. Current directory
.. Parent directory
~ Home directory
cd Move to home directory 
cd ..          # Move to parent directory
cd ~           # Move to home directory
cd /etc        # Move to absolute path
pwd            # Print current working directory

File Operations

mkdir dir1                  # Create directory
mkdir -p dir1/dir2/dir3     # Create full nested path
touch filename              # Create or update file

cp source dest              # Copy file
cp -r source_dir dest_dir   # Copy directory (must use -r)

mv file /tmp/               # Move file
mv old.txt new.txt          # Rename file

rm file1                    # Delete file
rm -r dir1                  # Delete directory
rm -rf dir1                 # Force delete — no confirmation
rmdir dir1                  # Remove empty directory only

Links

Inode Concept

Every file in Linux has an inode number — a unique identifier pointing to file data on disk. Hard links share the same inode. Soft links point to a path.

ls -i filename    # Check inode number of a file
Feature Soft Link (Symbolic) Hard Link
Command ln -s target linkname ln target linkname
Concept Pointer to a path Mirror — same inode
Original deleted Link breaks Data remains safe
Cross partition Yes No
Link directory Yes No
Verify ls -l shows → ls -i same inode

Shell Expansions

*           # Match everything:    rm *.txt
?           # Match one char:      ls file?.txt
[abc]       # Match from list:     ls file[123].txt
{1..20}     # Brace sequence:      touch file{1..20}.txt
{A,B,C}     # Brace list:          mkdir project{A,B,C}
$USER       # Variable expansion:  echo $USER
$(date)     # Command sub:         echo $(date)
" "         # Double quote: allows $, backticks
' '         # Single quote: everything literal

grep

grep "term" file          # Basic search
grep -i "term" file       # Case insensitive
grep -v "term" file       # Invert match
grep -r "term" folder/    # Recursive
grep -w "word" file       # Whole word only
grep "^root" /etc/passwd  # Lines starting with root
grep "bash$" /etc/passwd  # Lines ending with bash

# Combine commands with pipes:
grep "error" logfile | sort | head    # Filter → sort → top 10

Viewing & Sorting

head -n 5 file    # First 5 lines
tail -n 5 file    # Last 5 lines
tail -f file      # Real-time follow (logs)
sort file         # Alphabetical sort
sort -n file      # Numeric sort
sort -u file      # Remove duplicates
CH-3
Users & Groups
useradd, usermod, sudo, /etc/shadow, chage·Intermediate
▶

User Types

Type UID Range Description
Superuser (root) 0 Full system control
System User 1–999 Used by services/daemons
Regular User 1000–60000 Normal login users 
id username    # Check UID, GID and group memberships

Primary vs Secondary Groups

  • Primary Group — Default group assigned when user is created. New files created by the user belong to this group.
  • Secondary Group — Additional groups a user can join for extra access permissions.
id root        # Check root's UID, GID and groups
id username    # Check any user's UID, GID and group memberships

/etc/passwd Format

username : x : UID : GID : /home/dir : /bin/shell
# x = password stored in /etc/shadow

Create Users

useradd username       # Create new user
passwd username        # Set password
useradd --help         # View all options

usermod — Modify User

usermod -l newname oldname           # Rename
usermod -g groupname username        # Change primary group
usermod -aG groupname username       # Add secondary group
usermod -aG group1,group2 username   # Multiple groups
usermod -d /new/home username        # Change home dir
usermod -d /new/home -m username     # Change + move files
usermod -s /bin/bash username        # Change shell
usermod -L username                  # Lock account
usermod -U username                  # Unlock account
usermod -u 3005 username             # Change UID
usermod -e YYYY-MM-DD username       # Set expiry date
usermod -f days username             # Set inactive days after password expiry
usermod -c "Full Name" username      # Set comment
⚠️ Always use -a with -G. Without -a, all existing group memberships are overwritten.

groupmod

groupmod -n newgroup oldgroup    # Rename group
groupmod -g newGID groupname     # Change GID
# Group info: /etc/group

SUDO

visudo    # Edit sudoers safely

username ALL=(ALL) ALL           # Grant sudo to user
%groupname ALL=(ALL) ALL         # Grant sudo to group
username ALL=(ALL) NOPASSWD:ALL  # Skip password prompt

Switch Users

su username       # Switch (no env load)
su - username     # Switch with full environment
su -              # Switch to root (needs root password)
sudo -i           # Switch to root using your own password

/etc/shadow — Password Storage

All encrypted passwords are stored in /etc/shadow. Never edit this file directly.

root:$6$hashvalue:19130:0:99999:7:::
Field Meaning
Encrypted Password Hashed password value
Last Password Change Days since Jan 1, 1970
Minimum Age Days before password can be changed
Maximum Age Password validity duration
Warning Days Days before expiry to warn user
Inactive Days Grace period after expiry
Expiry Date Account expiry date

Password Policy

chage username       # Interactive policy editor
chage -l username    # List current policy
chage -d 0 username  # Force password change at next login
date -d "240 days"   # Calculate future date
# Future user defaults: /etc/login.defs
# Defines: PASS_MAX_DAYS, PASS_MIN_DAYS, PASS_WARN_AGE
💡 Always verify changes using id username and chage -l username after modifications.
CH-4
File Access Control — DAC
chmod, chown, SUID, SGID, sticky bit, umask·Intermediate
▶

Access Control Models

Model Name How it Works
DAC Discretionary Access Control File owner decides who can access. Standard rwx permissions (chmod/chown).
MAC Mandatory Access Control System-wide policies enforced by SELinux. Controls how processes interact with resources regardless of standard permissions.
💡 Both DAC and MAC must allow access. SELinux (MAC) adds an extra layer on top of standard permissions (DAC).

Reading Permissions

ls -l filename

-rwxrwxrwx
 │││││││││
 ││││││└└└── Others (r,w,x)
 │││└└└───── Group  (r,w,x)
 └└└──────── User   (r,w,x)

Permission Meanings

Permission On a File On a Directory
r (read) View file content List directory contents (ls)
w (write) Modify file content Create/delete files inside
x (execute) Run as program/script Enter directory (cd into it)

Symbolic Method

chmod u+x file       # Add execute for user
chmod g-w file       # Remove write from group
chmod o=r file       # Set others to read only
chmod a+x file       # Add execute for all

# Set all at once:
chmod u=rwx,g=rx,o=r file    # User=rwx, Group=r-x, Others=r--

Octal Method

Permission Value
read (r) 4
write (w) 2
execute (x) 1
all (rwx) 7 
chmod 755 file    # rwxr-xr-x
chmod 644 file    # rw-r--r--
chmod 700 file    # rwx------ (only owner has full access)
chmod 777 file    # rwxrwxrwx

Ownership

chown user file            # Change user ownership
chown user:group file      # Change user and group
chown :group file          # Change group only (colon syntax)
chgrp groupname file       # Change group only (alternative)

Special Permissions

SUID (runs as file owner)

chmod u+s file    # Set SUID
chmod 4744 file   # Set SUID (octal prefix 4)
chmod u-s file    # Remove SUID
chmod 0744 file   # Remove SUID (octal)

SGID — on Files vs Directories

  • On files — File runs with the group owner's privileges
  • On directories — New files created inside inherit the directory's group ownership
chmod g+s path    # Set SGID
chmod 2744 file   # Set SGID (octal prefix 2)
chmod g-s path    # Remove SGID

Sticky Bit — Real World Example

ls -ld /tmp       # /tmp uses sticky bit — users can't delete others' files
chmod o+t path    # Set sticky bit
chmod 1744 file   # Set sticky bit (octal prefix 1)
chmod o-t path    # Remove sticky bit
⚠️ In production: avoid 777, use SGID for collaborative directories, use sticky bit for shared writable directories like /tmp.

Default Permissions — umask

umask             # Check current umask
# Default perm = Max - umask (file max=666, dir max=777)

umask 027         # Set temporarily
# User-specific: add to ~/.bashrc
# Global: add to /etc/profile
💡 User-specific umask always takes priority over the global setting.
CH-6
Processes, Monitoring & Task Scheduling
ps, top, kill, nice, crontab, at, tuned·Intermediate
▶

Process Types

  • Foreground — Attached to terminal. Example: vim file.txt
  • Background — Detached. Append &. Example: firefox & — shows [jobID] PID

Job Control

command &     # Start in background
jobs          # List shell jobs
fg %1         # Bring job 1 to foreground
bg %1         # Resume stopped job in background
Ctrl-Z        # Suspend current foreground process
disown %1     # Remove job from shell table

Process Listing

ps                         # Processes in current terminal
ps -l                      # Long format (STAT, PPID, PRI, NI)
ps aux                     # All processes, all users
ps -o pid,ppid,uid,cmd     # Custom columns
pgrep firefox              # Find PID by name
pidof sshd                 # PID of a service
pstree                     # Process tree
top                        # Real-time viewer
htop                       # Enhanced real-time viewer

Process States

State Meaning
R Running or runnable
S Sleeping (waiting for event)
D Uninterruptible sleep (I/O wait — cannot be killed)
T Stopped or traced
Z Zombie — finished execution but parent hasn't collected exit status. Remains in process table until parent reads it.

ps aux — Key Columns

Column Meaning
USER Process owner
PID Process ID
%CPU CPU usage percentage
%MEM Memory usage percentage
STAT Process state (R, S, D, T, Z)
COMMAND Command that started the process

Signals & Kill

kill <PID>             # SIGTERM — polite stop
kill -9 <PID>          # SIGKILL — immediate force kill
kill -SIGSTOP <PID>    # Suspend process
kill -SIGCONT <PID>    # Resume suspended process
kill -l                 # List all signal names
pkill processname       # Kill by name
killall processname     # Kill all with exact name

Load Monitoring

uptime     # Shows: current time, uptime, load averages
# Load average: 5.20, 4.30, 3.10
#               1min  5min  15min
# → 1 min = most recent   |  15 min = longer trend

lscpu      # CPU count — check CPU(s): field
💡 Formula: Load Average ÷ Number of CPUs
Example: Load=5.2, CPUs=4 → 5.2÷4 = 1.3 → system is ~30% overloaded.
If result > 1 → CPU demand exceeds capacity.

Priority — nice & renice

Range: -20 (highest) to +19 (lowest). Default = 0.

nice -n 10 command         # Start with lower priority
renice -n -5 -p 12345      # Change running process priority
ps -l                       # View NI column
⚠️ Only root can set negative nice values.

Tuned — Performance Profiles

tuned-adm active                         # Check active profile
tuned-adm recommend                      # System-suggested profile
tuned-adm profile virtual-guest          # Apply virtual-guest profile
tuned-adm profile throughput-performance # Apply throughput profile
tuned-adm profile balanced               # Apply balanced profile
systemctl status tuned.service           # Check tuned is running
# Config:    /etc/tuned/tuned-main.conf
# Profiles:  /usr/lib/tuned/
💡 Admin tips: Always try SIGTERM before SIGKILL. Use systemctl status sshd to check service process details. Monitor load average regularly on production systems.

at — One-Time Scheduling

💡 Requires atd service: systemctl enable atd --now
at 07:40              # Schedule at specific time
at now + 5 minutes    # Relative scheduling
at 14:00              # 24-hour format
at tomorrow           # Tomorrow same time
at 10:00 Jan 25       # Specific date
atq                   # View pending jobs
atrm 1                # Remove job ID 1

crontab — Recurring Scheduling

💡 Requires cronie package and crond service: systemctl enable crond --now
# Cron syntax:
# ┌─────── Minute     (0-59)
# │ ┌───── Hour       (0-23)
# │ │ ┌─── DayOfMonth (1-31)
# │ │ │ ┌─ Month      (1-12)
# │ │ │ │ ┌ DayOfWeek (0-7, Sun=0 or 7)
# * * * * * command

crontab -e           # Edit cron jobs
crontab -l           # List cron jobs
crontab -r           # Remove ALL cron jobs
crontab -u student -e # Edit another user's crontab (as root)

# Every minute:
* * * * * echo "hello" >> /tmp/cron.txt

# Every 2 minutes (/ = interval):
*/2 * * * * echo "task running"

# Between 5 PM-7 PM on Wednesday (- = range):
0 17-19 * * 3 echo "evening task"

# Tuesday and Thursday at 4 PM (, = multiple values):
0 16 * * 2,4 echo "weekly task"

# System-wide /etc/crontab needs a USERNAME field:
* * * * * student echo "hello" >> /tmp/hello.txt

# Always redirect output for debugging:
* * * * * /scripts/backup.sh >> /var/log/backup.log 2>&1

# Access control:
/etc/cron.allow    # Only listed users permitted
/etc/cron.deny     # Listed users blocked
💡 Always use absolute paths in cron jobs. Check logs: journalctl -u crond

Anacron — Catch Missed Jobs

Anacron ensures scheduled jobs run even if the system was powered off at the scheduled time. Useful for laptops and systems not running 24/7.

vim /etc/anacrontab    # Anacron configuration

# Place scripts in these directories for automatic execution:
/etc/cron.daily/      # Run daily
/etc/cron.weekly/     # Run weekly
/etc/cron.monthly/    # Run monthly
CH-8
Install & Update Software Packages
dnf, yum, rpm, repositories·Beginner
▶

What is a Package?

A package is a bundled collection of: binary files, libraries, configuration files, dependencies, and documentation. Example: httpd contains the Apache web server binary, config files, and modules.

Package Managers

  • rpm — Low-level, installs .rpm files directly, no dependency resolution
  • yum — Package resolver (older, RHEL 7)
  • dnf — Current default in RHEL 9, faster and more efficient

Repositories: BaseOS (essential system packages) | AppStream (additional apps, dev tools, languages)

Repository files (.repo extension) stored in: /etc/yum.repos.d/

Core DNF Commands

dnf install packagename         # Install
dnf remove packagename          # Remove
dnf info packagename            # View package info
dnf list packagename            # List specific package
dnf list installed              # List all installed packages
dnf list                        # List all available packages
dnf search all "keyword"        # Search by keyword
dnf provides commandname        # Find what package provides a command
dnf history                     # View transaction history
dnf history undo 5              # Undo transaction ID 5
dnf update                      # Update all (keeps obsolete packages)
dnf upgrade                     # Update + remove obsolete packages

Package Groups

dnf group list                  # List all package groups
dnf group info "Web Server"     # Info about a group
dnf group install "Web Server"  # Install a group
# Package Group = collection of related packages
# Environment Group = collection of multiple groups

RPM Direct Commands

rpm -ivh package.rpm    # Install .rpm file (i=install v=verbose h=progress)
rpm -q httpd            # Query if package is installed

Repository Management

dnf repolist             # Check all configured repositories
dnf repolist enabled     # Show only enabled repos

# CLI method:
dnf config-manager --add-repo="http://repo_url"

# Manual method — create .repo file:
vim /etc/yum.repos.d/myrepo.repo

[myrepo]
name=My Repo
baseurl=http://your_repo_url/
enabled=1       # 1=enabled, 0=disabled
gpgcheck=0      # 0=skip GPG signature check (not recommended in production)
CH-9
Basic Storage & Partitions
fdisk, mkfs, mount, fstab, swap·Intermediate
▶

Storage Types

Type Usage Notes
Block-Level Linux partitions, filesystems, swap Divides storage into fixed-size blocks. High performance. Used in RHEL.
Object-Level Cloud environments (S3, etc.) Stores data as objects. Not used for local Linux partition management.

A partition is a defined logical section of a disk. Example: a 500GB disk split into 100GB for OS, 200GB for videos, 200GB for documents.

Check Storage

lsblk           # List block devices and partitions
lsblk -f        # Show filesystem type and UUID
lsblk -fp       # Full path + filesystem type + UUID
df -h           # Disk usage of mounted filesystems
free -h         # RAM and swap usage

Partition Schemes

Scheme Max Partitions Max Size Type
MBR (dos) 15 2 TB Oldest, BIOS/UEFI
GPT 128 8 Zettabytes New, UEFI

Create Partition with fdisk

Two common tools: fdisk (standard, MBR/GPT) | parted (advanced, for large disks)

fdisk /dev/sdb     # Open disk
# n = new partition
# p = primary (Enter for default)
# Enter = accept default first sector
# +2G = set size
# p = print/verify
# w = write and exit

partprobe          # Notify kernel

Format & Mount

mkfs.xfs /dev/sdb1         # Format as XFS
mkfs.ext4 /dev/sdb1        # Format as EXT4
mkfs -t xfs /dev/sdb1      # Alternative syntax

mkdir /mnt/data1
mount /dev/sdb1 /mnt/data1    # Temporary mount
df -h                          # Verify

Filesystem Types

Filesystem Usage
XFS Default in RHEL 9, high performance, cannot be reduced
ext4 General purpose, supports reduce
FAT32 USB drives (max file size 4GB)
NTFS Windows filesystem

Permanent Mount — /etc/fstab

vim /etc/fstab
/dev/sdb1  /mnt/data1  ext4  defaults  0  0

systemctl daemon-reload
mount -a        # Test (no output = success)
df -h           # Verify
💡 In production, use UUID instead of device name in /etc/fstab — device names can change after reboot but UUID never changes. Get UUID with: lsblk -f

Complete Storage Workflow

# 1. Detect disk:     lsblk
# 2. Create partition: fdisk → n → size → w → partprobe
# 3. Format:          mkfs.xfs /dev/sdXN
# 4. Mount point:     mkdir /mnt/data
# 5. Temporary mount: mount /dev/sdXN /mnt/data
# 6. Make permanent:  vim /etc/fstab → add entry
# 7. Test:            systemctl daemon-reload && mount -a
# 8. Verify:          df -h
# C - Create:  fdisk → n → +1G → t → 82(MBR)/8200(GPT) → w
# F - Format:  mkswap /dev/sdXN
# P - Persist: /etc/fstab: /dev/sdXN swap swap defaults 0 0
# A - Activate: swapon -a
swapon --show    # Verify
CH-10
LVM — Logical Volume Manager
PV → VG → LV, extend, reduce, delete·Advanced
▶

LVM Hierarchy

Disk → Partition → PV → VG → LV → Filesystem → Mount Point

Physical Volume (PV) → Volume Group (VG) → Logical Volume (LV)

Physical storage cannot be resized. Logical storage (LVM) is flexible — it can be extended and (for EXT only) reduced.

💡 Memory code: PPVLMF — Partition → PV → VG → LV → Mount → Filesystem

Part A — Create LVM

# 1. Create LVM partition
fdisk /dev/sdb  →  n → Enter → +1G → t → 8e(MBR) → w
partprobe

# 2. Physical Volume
pvcreate /dev/sdb1
pvdisplay

# 3. Volume Group
vgcreate myvg /dev/sdb1
vgcreate -s 16M myvg /dev/sdb1    # Custom PE size
vgdisplay

# 4. Logical Volume
lvcreate -n mylv -L 5G myvg       # By size (-L = human readable size)
lvcreate -n mylv -l 255 myvg      # By PE count (-l = number of extents)
lvdisplay                          # Verify logical volume

# 5. Filesystem
mkfs.xfs /dev/myvg/mylv

# 6. Mount
mkdir /mnt/data
vim /etc/fstab
# /dev/myvg/mylv  /mnt/data  xfs  defaults  0  0
mount -a
df -h

Part B — Extend

lvextend -L +1G /dev/myvg/mylv         # Extend by size
lvextend -l +255 /dev/myvg/mylv        # Extend by PE count
lvextend -L +500M -r /dev/myvg/mylv    # -r also resizes filesystem together

xfs_growfs /dev/myvg/mylv              # Resize XFS manually
resize2fs /mnt/data                     # Resize EXT manually

# If VG is full — add new PV first:
pvcreate /dev/sdc1
vgextend myvg /dev/sdc1
lvextend -L +5G -r /dev/myvg/mylv
💡 Always check VG free space before extending: vgdisplay

Part C — Reduce

⚠️ XFS cannot be reduced. Only EXT2/3/4 supports reduction. Always backup before reducing. 
umount /mnt/data
fsck /dev/myvg/mylv
resize2fs /dev/myvg/mylv 3G
lvreduce -L 3G /dev/myvg/mylv
mount -a

# Shortcut method (EXT only):
lvreduce -L -512M -r /dev/myvg/mylv    # -r handles filesystem resize too

Part D — Delete (Reverse Order)

umount /mnt/data
vim /etc/fstab              # Remove entry
systemctl daemon-reload
lvremove /dev/myvg/mylv
vgremove myvg
pvremove /dev/sdb1

What is a Service / Daemon?

A service is a program running in the background. When running, it becomes a process. Most Linux background services end with d (daemon).

Service Purpose
sshd Secure remote login
crond Scheduled tasks
chronyd Time synchronization
httpd Web server
tuned Performance tuning

systemd is the system and service manager. It is always the first process started during boot — always PID 1.

systemd Unit Types

Unit Type Purpose
service Manages background services
socket Manages socket activation
target Groups other units (like runlevels)
mount Controls mount points
timer Replaces cron jobs
path Monitors file paths
slice Controls resource allocation

systemd Service Management

systemctl start httpd              # Start service
systemctl stop httpd               # Stop service
systemctl status httpd             # Check status
systemctl enable httpd             # Enable at boot
systemctl disable httpd            # Disable at boot
systemctl enable httpd --now       # Enable + start immediately
systemctl disable httpd --now      # Disable + stop immediately
systemctl restart httpd            # Stop + start
systemctl reload httpd             # Reload config (no stop)
systemctl reload-or-restart httpd  # Let system decide
systemctl mask httpd               # Block from starting at all
systemctl unmask httpd             # Remove block

Practical Example — httpd

dnf install httpd                         # Install web server
vim /etc/httpd/conf/httpd.conf            # Edit config file
# Default document root: /var/www/html/
# To change port: find "Listen 80" → change to "Listen 8080"
systemctl restart httpd                   # Apply changes

Targets — System Modes

Target Purpose
multi-user.target CLI mode (no GUI)
graphical.target GUI mode (desktop) 
systemctl get-default                        # Check default
systemctl set-default multi-user.target      # Set CLI default (reboot needed)
systemctl set-default graphical.target       # Set GUI default (reboot needed)
systemctl isolate multi-user.target          # Switch to CLI now (temporary)
systemctl isolate graphical.target           # Switch to GUI now (temporary)
systemctl list-dependencies httpd            # Show dependencies

Linux Boot Sequence

  1. BIOS/UEFI — Performs POST (Power On Self Test), hardware check, finds bootloader on disk
  2. GRUB2 — Shows boot menu, loads kernel + initramfs
  3. Kernel — Detects hardware, starts systemd (PID 1)
  4. initramfs — Temporary root filesystem for early boot
  5. systemd — Reads default target, starts services
  6. Login — CLI or GUI based on target

Root Password Recovery

This is a standard Linux administration procedure documented in Red Hat's public documentation.
# 1. Reboot: reboot
# 2. At GRUB menu: press Up/Down arrow to stop countdown
# 3. Select boot entry, press 'e' to edit
# 4. Find line starting with 'linux', press End key
# 5. Add at end of that line:
rd.break
# 6. Press Ctrl+X to boot into emergency shell

# 7. Remount filesystem as read-write:
mount -o remount,rw /sysroot

# 8. Enter real system:
chroot /sysroot

# 9. Change password:
passwd

# 10. Create SELinux relabel marker:
touch /.autorelabel

# 11. Exit and reboot:
exit
exit
⚠️ The touch /.autorelabel step is required. Without it the system may fail to boot after a password change.

What is a Log?

A log is a record of events that occur in a system — from kernel messages and service activity to security events and user actions. Logs are essential for troubleshooting, security auditing, and performance analysis.

Logging Architecture

systemd-journald captures raw events (kernel, boot, services) → passes to rsyslog → rsyslog writes persistent files to /var/log/

Service Storage Format Survives Reboot?
systemd-journald /run/log/journal Binary (use journalctl) No (volatile by default)
rsyslog /var/log/ Plain text Yes (persistent)

journalctl — Read Journal Logs

journalctl                  # View all logs
journalctl -n 20            # View last 20 entries
journalctl -f               # Follow live logs (like tail -f)
journalctl -u sshd          # Logs for specific service
journalctl -b               # Logs since last boot
journalctl | grep sealert   # Filter for SELinux denials

Key Log Files

File Contents
/var/log/messages General system messages
/var/log/secure Authentication, SSH, sudo
/var/log/maillog Mail server activity
/var/log/cron Cron job execution
/var/log/boot.log Boot/startup messages 
tail -f /var/log/messages    # Real-time monitoring
less /var/log/secure         # Browse log file

rsyslog Configuration

Logs follow the format: facility.priority

  • Facility — Source of log (kern, mail, cron, auth)
  • Priority — Severity level (debug, info, warning, err, crit, emerg)
# Examples:
kern.debug     # Kernel debug messages
kern.crit      # Kernel critical messages

# Config file:
/etc/rsyslog.conf

# After changes, restart:
systemctl restart rsyslog.service

logger — Generate Logs Manually

logger -p kern.debug "My custom test message"
# Useful for testing log configurations

Make Journal Persistent

By default journald logs are volatile (lost on reboot). To make permanent:

vim /etc/systemd/journald.conf
# Change Storage= value:
# Storage=auto       → persistent if directory exists
# Storage=volatile   → memory only
# Storage=persistent → always store permanently

Storage=persistent

systemctl restart systemd-journald

NTP with chrony

# 1. Set timezone:
timedatectl                              # Check current settings
timedatectl list-timezones               # List all available timezones
timedatectl set-timezone Asia/Kolkata    # Set timezone
tzselect                                 # Interactive finder (suggests but does NOT apply automatically)

# 2. Install:
dnf install chrony -y

# 3. Configure:
vim /etc/chrony.conf
# Add (comment out existing pool/server lines):
server 172.25.254.254 iburst    # iburst = fast initial sync

# 4. Apply:
systemctl restart chronyd
systemctl enable chronyd

# 5. Verify:
chronyc sources
# * = synchronized correctly
# ? = not synchronized / problem

# Disable NTP and set time manually:
timedatectl set-ntp false
timedatectl set-time "2025-11-03 10:30:00"

SELinux — Security Enhanced Linux
Contexts, ports, booleans, semanage·Advanced
▶

What is SELinux?

SELinux (Security-Enhanced Linux) is a kernel-level security module originally developed by the NSA (National Security Agency). It implements Mandatory Access Control (MAC) using context labels. Both DAC (chmod) and MAC (SELinux) must allow access.

SELinux Context — 4 Parts

Every file, process and port has a context label in this format:

unconfined_u:object_r:user_home_t:s0
Field Example Meaning
SELinux User unconfined_u SELinux user identity
Role object_r Role-based access control
Type user_home_t Most important — used in policy decisions
Level s0 MLS/MCS sensitivity level

Status & Modes

sestatus          # Detailed status
getenforce        # Quick mode check
# Modes:
# Enforcing  — policies applied
# Permissive — inactive but violations logged
# Disabled   — completely off

setenforce 1      # Enforcing (temporary)
setenforce 0      # Permissive (temporary)
vim /etc/selinux/config    # Permanent change (requires reboot)

View Context Labels

ls -Z filename       # File context
ls -dZ /tmp/         # Directory context
ps axZ               # Process contexts

Change Context — Temporary

chcon -t httpd_sys_content_t demo.html

Change Context — Permanent

# Step 1: Add to SELinux policy database
semanage fcontext -a -t httpd_sys_content_t "/root/demo.html"

# Use wildcard for entire directory:
semanage fcontext -a -t httpd_sys_content_t "/webdata(/.*)?"

# Step 2: Apply from database
restorecon -v /root/demo.html     # Single file
restorecon -Rv /webdata/          # Recursive directory

# View the database:
semanage fcontext -l | grep webdata

Apache Practical Example

# Install web server:
dnf install httpd -y

# Default document root:
/var/www/html/

# Start and enable:
systemctl enable httpd --now

# Default context type for web content:
httpd_sys_content_t

# If file has wrong context, web server returns 403 Forbidden
# Fix: semanage fcontext -a -t httpd_sys_content_t "/path/file"
#       restorecon -Rv /path/

Port Contexts

semanage port -l                    # List all ports
semanage port -l | grep http        # Filter by type
semanage port -a -t http_port_t -p tcp 82    # Add port
systemctl restart httpd
curl localhost:82                    # Test

Troubleshooting & Booleans

journalctl | grep sealert                     # Find denials
getsebool -a | grep http                      # List booleans
setsebool httpd_enable_homedirs on            # Set temporarily
setsebool httpd_enable_homedirs on -P         # Set permanently

Firewall Management
firewall-cmd, zones, services, ports·Intermediate
▶

What is a Firewall?

A firewall controls incoming and outgoing network traffic based on defined rules. In modern Linux, firewall management is done using the firewalld service, managed with firewall-cmd.

Zones

A zone defines the trust level of a network connection. Each zone has different default rules.

Zone Trust Level
trusted All connections accepted
home / work Mostly trusted, some services allowed
internal Internal network, higher trust
public Default — untrusted public network
dmz Limited access, for server exposure
block All connections rejected
drop All connections silently dropped
💡 General pattern: --get-... to view | --set-... to configure | --add-... to allow | --remove-... to deny
firewall-cmd --get-services                         # List services
firewall-cmd --get-default-zone                     # View default zone
firewall-cmd --set-default-zone=public              # Set default zone

firewall-cmd --add-service=http --permanent
firewall-cmd --remove-service=http --permanent

firewall-cmd --add-port=8080/tcp --permanent
firewall-cmd --remove-port=8080/tcp --permanent

firewall-cmd --add-source=192.168.1.0/24 --zone=internal
firewall-cmd --remove-source=192.168.1.0/24 --zone=internal

firewall-cmd --reload        # Apply permanent rules
firewall-cmd --list-all      # View all active rules
⚠️ Without --permanent, changes are lost on reboot. After using --permanent, run --reload to apply immediately.

Networking & NFS

IP Address Basics

IPv4 — 4 octets, 32 bits total, each octet ranges 0–255. Example: 192.168.1.24

Subnet — Example: 192.168.0.0/16 — /16 = network bits, remaining = host bits. A network is a pool of IP addresses divided into network portion and host portion.

Network Commands

ip addr                      # Show IP addresses (modern)
ip a                         # Short alias for ip addr
ifconfig                     # Older alternative (may need net-tools)
ip link show                 # List network interfaces + MAC + status
ip route                     # Show routing table / default gateway
ping hostname                # Test connectivity
ping google.com -c 5         # Limit to 5 replies
tracepath hostname            # Trace packet route — each hop = one router
ss -t                        # Show TCP connections
ss -ta                       # All TCP connections including listening
ss -l                        # Listening ports only
hostname                     # Show current hostname

DNS Resolution

When you type a domain name, the system checks /etc/hosts first. If not found, it queries the DNS server to resolve to an IP address.

cat /etc/hosts    # View local hostname resolution file

# Add custom entry:
192.168.1.10   myserver     # Now 'myserver' resolves to 192.168.1.10

ss Output Explained

Column Meaning
Local Address Your system's address and port
Peer Address Remote system address and port
State Connection status (ESTAB, LISTEN, etc.)
Command Purpose
w Shows logged-in users
ss Shows active network connections

Configure Network — nmtui

nmtui    # Launch Terminal UI for network config
# Steps:
# 1. Edit a connection → select interface (e.g., eth0)
# 2. IPv4 Configuration → change Automatic to Manual
# 3. Enter IP address, gateway, DNS
# 4. Back → Set System Hostname
# 5. Activate a connection → deactivate + reactivate

ping <gateway_IP>    # Verify connectivity
yum list             # Verify repository access

NFS — Network File System

NFS allows a client to access files shared by a server over a network as if they were local files. Used in enterprise and centralized storage environments.

dnf install nfs-utils -y    # Install NFS client utilities
showmount --exports server_hostname    # Check what server is sharing

mkdir /mountpoint    # Create local mount point

# Temporary mount:
mount -t nfs -o rw,sync server:/export /mountpoint
# -t nfs = filesystem type | rw = read-write | sync = synchronous write

# Permanent (fstab):
server:/export  /mountpoint  nfs  rw,sync  0  0
mount -a        # Test entry
umount /mountpoint    # Unmount when needed

AutoFS — On-Demand Mounting

AutoFS mounts shares only when accessed. Service ON = visible when accessed. Service OFF = not available. 

dnf install autofs -y
⚠️ Do NOT edit /etc/auto.master directly. Create files inside /etc/auto.master.d/ instead.

Type 1 — Direct (base mount point already exists)

vim /etc/auto.master.d/direct.autofs
# Add:  /-    /etc/auto.direct

vim /etc/auto.direct
# Add:  /public   -rw,sync   server:/sharing_directory

systemctl restart autofs
# Accessing /public → triggers automatic mount

Type 2 — Indirect (base mount point does NOT exist)

vim /etc/auto.master.d/indirect.autofs
# Add:  /guest    /etc/auto.indirect

vim /etc/auto.indirect
# Add:  mount   -rw,sync   server:/sharing_directory

systemctl restart autofs
# cd /guest/mount → mounts automatically

Type 3 — Indirect with Wildcard (multiple subdirs)

vim /etc/auto.master.d/wildcard.autofs
# Add:  /mnt/mount    /etc/auto.wildcard

vim /etc/auto.wildcard
# Add:  *   -rw,sync   server:/shares/public/&
# * = wildcard | & = substitutes the matched directory name

systemctl restart autofs
# cd /mnt/mount/one → mounts server:/shares/public/one

NFS Mounting Types Summary

Type Method Persistence When Used
Manual mount command Temporary Quick access
Persistent /etc/fstab At boot Permanent mount
AutoFS autofs service On-demand Large environments

Podman — Container Management
Images, run, volumes, port forwarding, services, rootless·Advanced

Why Containers?

Every application needs: binary files + libraries + dependencies. When multiple apps share libraries, changing one can break others.

Feature Virtual Machine (VM) Container
OS Full OS per VM Shares host kernel
Resources Heavy (dedicated CPU/RAM) Lightweight
Startup Minutes Seconds
Isolation Strong Good

A container image contains: base OS + required libraries + application + predefined configuration. Images are stored in registries (public like quay.io, or private).

# Registry configuration:
/etc/containers/registries.conf

Install & Search

dnf install container-tools     # Install podman
podman -v                        # Verify version
podman search httpd              # Search registries
# Registry config: /etc/containers/registries.conf

Images

podman pull IMAGE_URL      # Download image
podman images             # List local images
podman rmi IMAGE_URL      # Remove image

Containers

podman run -d --name webserver IMAGE_URL
# -d = detached (background)
# --name = container name

podman ps          # Running containers
podman ps -a       # All containers (running + stopped)
podman stop webserver
podman rm webserver
podman rm webserver -f    # Force remove

Execute & Copy

podman exec -it webserver bash      # Enter container shell
cd /var/www/html/
curl localhost                       # Test web content
exit                                 # Leave container

podman cp index.html webserver:/var/www/html/

Port Forwarding

Port forwarding must be set at container creation — it cannot be added later.
podman run -d --name webserver -p 8080:80 IMAGE_URL
curl localhost:8080

Volumes

podman run -d --name webserver \
  -v /backup/:/var/www/html/:Z \
  -p 8080:80 IMAGE_URL
# :Z = SELinux context matching

Build Local Image

vim containerfile
# FROM base-image
# RUN commands
# CMD ["default-command"]

podman build -t myimage -f containerfile
podman run -d --name myapp localhost/myimage:latest

Container as systemd Service (Root)

cd /etc/systemd/system
podman generate systemd --name webserver --files
mv container-webserver.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable container-webserver.service --now
systemctl status container-webserver.service

Root vs Rootless Containers

Feature Root Container Rootless Container
Runs as root user Normal user
Privileged ports Yes (<1024) No
Security Less secure More secure

Rootless Containers

⚠️ SSH directly as the target user. su - username may not behave correctly for rootless container operations.
w    # Verify correct user session

mkdir -p ~/.config/systemd/user
cd ~/.config/systemd/user
podman generate systemd --name myapp --files
mv container-myapp.service ~/.config/systemd/user/
systemctl --user daemon-reload
systemctl --user enable container-myapp.service --now
systemctl --user status container-myapp.service
🔑 To start container at system boot (not just at user login):

loginctl enable-linger username

Without this, the container only starts when the user logs in interactively.

tar — Archive & Compress

Flag Compression Extension
-j bzip2 .tar.bz2
-z gzip .tar.gz
-J xz .tar.xz 
tar -cvjf backup.tar.bz2 /usr/local     # Create bzip2 archive of /usr/local
tar -cvzf backup.tar.gz /usr/local      # Create gzip archive
tar -cvJf backup.tar.xz /usr/local      # Create xz archive
# -c=create  -v=verbose  -f=filename

tar -xvf archive.tar.bz2               # Extract archive (-x = extract)
tar -tvjf backup.tar.bz2               # View contents WITHOUT extracting (-t = list)
man tar                                  # See all options

find — Locate Files

find / -name newfile                 # By exact name
find / -user jeel                    # By owner
find / -group devops                 # By group
find / -perm 750                     # By exact permission
find / -perm 750 -type d             # Directories only (-type d)
find / -perm 750 -type f             # Files only (-type f)
find / -perm /g+s                    # Files with SGID set
find / -size +30k                    # Files larger than 30KB
find / -size +30k -size -45k        # Between 30KB and 45KB

# Redirect output to file:
find / -user jeel > output.txt
cat output.txt                       # View results

# Execute command on found files:
find / -user jeel -exec cp -pvr {} /root/jeel-files/ \;
# {}  = placeholder for each found file
# \;  = end of exec command
# -p  = preserve permissions | -v = verbose | -r = recursive

Shell Script Basics

vim myscript.sh

#!/bin/bash
# ↑ Shebang — tells the system which interpreter to use (always required!)
find /usr -perm /g+s -size +30k -size -45k > /root/usr-files
# > redirect output to file

# Save and exit vim: :wq

chmod +x myscript.sh    # Give execute permission

sh myscript.sh          # Run using sh interpreter
./myscript.sh           # Run directly (needs execute permission)

cat /root/usr-files     # Verify output was saved correctly
💡 sudo !! re-runs your last command with sudo. !! = shell history expansion for the previous command.
